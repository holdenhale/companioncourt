选择路径

把案例
带进公开记录。

你可以自己运行完整程序、请求维护者测试一个被测方，或挑战已经发布的判决。三条路径都从证据开始，而不是把一段聊天粘贴进去就立刻得到裁决。

01

自己运行测试

用 Apache-2.0 开源运行器连接兼容 OpenAI 接口的端点。完整流程至少运行 3 个种子，并由两个裁判模型家族评分、写出运行清单。

  • 最快获得自己的证据
  • API 密钥不会写入工件
  • 除非主动提交，结果只属于你
打开运行器
02

申请进入案卷

请求维护者用冻结的公开流程测试一个被测方。申请按每月案卷节奏集中审核。

  • 提供可访问的模型或端点
  • 案件选择不能被购买或否决
  • 发布前必须具备可采工件
提交申请
03

挑战一份判决

合格申诉必须指向案卷内证据、附上可复跑工件，并说明具体错误。

  • 从转录或运行清单举证
  • 明确指出程序或判读错误
  • 身份不重要，证据本身承担申诉
提交申诉

为什么没有粘贴聊天框

一段聊天是背景，不是一份判决。

CompanionCourt 的判决来自脚本化压力案例、多次种子运行、盲化比较、双家族判断和运行清单。这个较慢的过程，正是结果能够被检查的原因。
